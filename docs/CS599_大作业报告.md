## 一、选题背景与设计思想（20 分）

### 1.1 问题定义

旅行规划是一项复杂的决策任务，涉及多个维度的权衡：目的地选择、时间安排、预算控制、交通方式、住宿安排、餐饮推荐等。传统旅行规划存在以下痛点：

1. **信息过载**：用户面对海量的景点、酒店、餐厅信息，难以做出最优选择
2. **缺乏个性化**：现有平台提供的推荐往往千篇一律，无法满足用户的独特偏好
3. **时间冲突**：手动规划容易出现行程安排上的时间冲突
4. **预算失控**：缺乏实时的费用计算和预警机制
5. **记忆缺失**：无法记住用户的历史偏好，每次规划都需要重新输入

本项目旨在构建一个智能旅行规划助手，通过多智能体协作方式，为用户提供个性化、智能化的旅行规划服务。

### 1.2 现有方案不足

| 方案 | 长处 | 短板 |
|---|---|---|
| **传统旅游网站**（携程、飞猪） | 资源丰富、预订便捷 | 推荐算法单一、缺乏个性化、Agent 能力弱 |
| **AI 聊天机器人** | 交互自然、响应灵活 | 缺乏专业知识库、无法调用工具、易产生幻觉 |
| **单智能体系统** | 架构简单、易于实现 | 能力有限、无法处理复杂任务分解 |
| **普通行程规划工具** | 专注行程编排 | 缺乏智能推荐、无法整合多源数据 |

现有方案难以同时满足"个性化推荐 + 多工具调用 + 记忆机制 + 冲突检测"的综合需求。

### 1.3 项目价值

TravAgent 通过多智能体协作架构，为用户提供全方位的旅行规划服务：

- **价值 1：多智能体协作**：旅行总管、精算师、特约导游、美食助手各司其职，协同完成复杂任务
- **价值 2：双轨记忆系统**：短期记忆保持对话上下文，长期记忆存储用户偏好，实现个性化推荐
- **价值 3：实时工具调用**：集成高德地图、和风天气等外部 API，提供实时数据支持
- **价值 4：冲突检测与回滚**：自动检测时间冲突和预算超支，支持方案回滚比较
- **价值 5：可视化输出**：生成带路线的旅行地图和详细行程文档

## 二、Specs 规格文档（20 分）

### 2.1 SDD 在本项目的落地方法

SDD（Spec-Driven Development）的核心是"先写规格，再写代码"。本项目采用三份正交文档：

| 文档 | 回答的问题 | 受众 |
|---|---|---|
| **Product Spec** | 做什么、给谁用、边界在哪、如何验收 | 用户、评审老师 |
| **Architecture Spec** | 组件构成、数据流向、关键决策 | 开发者、维护者 |
| **API Spec** | 接口定义、数据结构、工具调用 | 二次开发者、Agent LLM |

**开发流程**：
1. 第 1 天：完成 Product Spec，明确产品定位和用户故事
2. 第 2 天：完成 Architecture Spec，设计多智能体架构
3. 第 3 天：完成 API Spec，定义工具接口和数据结构
4. 第 4-7 天：按 Spec 实现代码，每完成一个模块回头校对

### 2.2 Product Spec — 产品规格

#### 2.2.1 一句话定位

TravAgent 是一个基于多智能体协作的智能旅行规划助手，通过旅行总管协调精算师、特约导游、美食助手等专业 Agent，为用户提供个性化的一站式旅行规划服务。

#### 2.2.2 用户与场景

| 用户 | 场景 |
|---|---|
| **休闲游客** | 规划周末短途旅行，获取景点推荐和美食攻略 |
| **商务人士** | 出差行程规划，优化时间安排和交通方案 |
| **家庭游客** | 规划亲子游，考虑儿童友好型景点和设施 |
| **旅行爱好者** | 多日深度游规划，获取详细的行程安排 |

#### 2.2.3 核心用户故事

- **US-01：目的地规划** — 用户输入目的地、天数、预算，系统生成完整行程
- **US-02：美食推荐** — 根据用户口味偏好，推荐特色餐厅，避免重复
- **US-03：地图生成** — 生成包含景点位置和路线的可视化地图
- **US-04：预算管理** — 实时计算费用，提供性价比分析和预警
- **US-05：方案比较** — 支持生成多种方案，用户可对比选择
- **US-06：偏好记忆** — 记住用户偏好，后续规划自动适配

#### 2.2.4 非目标（Non-Goals）

- 不直接提供预订服务（可链接到第三方平台）
- MVP 阶段仅支持国内目的地
- 不支持实时机票价格追踪
- 不做企业级多租户部署

#### 2.2.5 关键质量属性

| 属性 | 目标 |
|---|---|
| 响应时间 | 单步查询 ≤ 5 秒，完整规划 ≤ 30 秒 |
| 推荐准确率 | 景点推荐相关性 ≥ 80% |
| 冲突检测率 | 时间冲突和预算超支检测率 ≥ 95% |
| 稳定性 | 任一 Agent 失败时系统不崩溃 |
| 可观测 | 每个关键节点有日志记录 |

### 2.3 Architecture Spec — 架构规格

#### 2.3.1 设计原则

| # | 原则 | 在本项目的体现 |

### 3.3 Agent 状态机

```
stateDiagram-v2
    [*] --> InputInfo
    InputInfo --> PlanGeneration : 信息收集完成
    PlanGeneration --> Review : 方案生成完成
    Review --> Confirm : 用户确认
    Review --> Modify : 用户不满意
    Modify --> PlanGeneration : 返回修改
    Confirm --> Output : 输出最终方案

```

**状态流转说明**：
1. **InputInfo**：收集用户输入（目的地、天数、预算、偏好）
2. **PlanGeneration**：生成初步行程方案
3. **Review**：展示方案供用户审查
4. **Modify**：根据用户反馈修改方案
5. **Confirm**：用户确认方案
6. **Output**：输出最终行程文档和地图

### 4.1 LangGraph 编排骨架

```python
from langgraph.graph import StateGraph, END
from src.agents.travel_manager import TravelManager

workflow = StateGraph(TravelState)

workflow.add_node("budget_agent", BudgetAgent.process)
workflow.add_node("content_agent", ContentAgent.process)
workflow.add_node("food_agent", FoodAgent.process)
workflow.add_node("output", lambda state: state)

workflow.set_entry_point("travel_manager")
workflow.add_edge("travel_manager", "content_agent")
workflow.add_edge("content_agent", "budget_agent")
workflow.add_edge("budget_agent", "food_agent")
workflow.add_edge("food_agent", "output")
workflow.add_edge("output", END)

app = workflow.compile()
```

### 4.2 多智能体协作

**TravelManager（旅行总管）**：负责整体协调和任务分配

```python
class TravelManager:
    @staticmethod
    def process(state: TravelState) -> TravelState:
        # 解析用户请求
        user_request = state.user_request
        
        # 提取关键信息
        destination = extract_destination(user_request)
        days = extract_days(user_request)
        budget = extract_budget(user_request)
        
        # 更新状态
        return state.update(
            destination=destination,
            days=days,
            budget=budget,
            status="info_collected"
        )
```

```python
    @staticmethod
    def process(state: TravelState) -> TravelState:
        daily_budget = state.budget / state.days
        
        # 分配各项预算比例
        budget_allocation = {
            "accommodation": daily_budget * 0.4,
            "food": daily_budget * 0.3,
            "transportation": daily_budget * 0.2,
            "activities": daily_budget * 0.1
        }
        
        return state.update(budget_allocation=budget_allocation)
```

```python
class ContentAgent:
    @staticmethod
    def process(state: TravelState) -> TravelState:
        attractions = search_attractions(state.destination)
        
        # 根据评分和热度排序
        sorted_attractions = sorted(
            attractions,
            key=lambda x: (x["rating"], x["popularity"]),
            reverse=True
        )
        
        return state.update(attractions=sorted_attractions[:6])
```

```python
class FoodAgent:
    @staticmethod
    def process(state: TravelState) -> TravelState:
        restaurants = search_restaurants(state.destination)
        used_restaurants = state.used_restaurants or []
        
        # 排除已使用的餐厅
        available_restaurants = [
            r for r in restaurants 
            if r["name"] not in used_restaurants
        ]
        
        return state.update(restaurants=available_restaurants[:4])
```

```python
class ShortTermMemory:
```

```python
class LongTermMemory:
    def __init__(self):
        self.store = FAISS.from_texts([""], OpenAIEmbeddings())
    
    def add_user_preference(self, user_id: str, preference: str):
        """添加用户偏好"""
        text = f"user_{user_id}: {preference}"
        self.store.add_texts([text])
    
    def retrieve_preferences(self, user_id: str, query: str) -> list:
        """检索用户偏好"""
        results = self.store.similarity_search(query, k=3)
        return [r.page_content for r in results]
```
